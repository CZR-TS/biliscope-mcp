# 更新日志 (Changelog)

所有关于 **Bilibili MCP Server** 的显著更改都将记录在此文件中。

---

## [1.3.7] - 2026-03-09

### 🚀 新增 (Added)
- **Cookie 过期智能检测机制**：为核心 API 客户端（`fetchWithWBI` 及 `fetchWithoutWBI`）增加了底层状态码拦截。当检测到由于凭证失效导致 B站接口返回 `-101`（未登录）时，工具会主动终止隐式降级并抛出明确的中文字幕提示 `COOKIE_EXPIRED`，大幅降低用户的 Debug 成本。
- *安全提示*：所有过期报错信息仅输出状态码和错误提示文本，**严格脱敏，绝不包含或打印真实的 Cookie 字符串**。

## [1.3.6] - 2026-03-09

### 🔧 修复 (Fixed)
- **字幕获取逻辑优化**：修复了在处理“充电专属”或“付费合集”视频时，由于 `need_login_subtitle` 标志位导致的程序硬性拦截 Bug。
  - **改进前**：程序检测到“需要登录查看字幕”标志后会立即中断，返回简介。
  - **改进后**：程序在登录状态下会无视该标志位，强制尝试获取 AI 字幕流。只要用户 Cookie 有效，即可成功解析充电视频的字幕内容。

### 📝 文档 (Documentation)
- 新增了详尽的 Bug 修复记录，涵盖了技术层面的根因分析（Root Cause Analysis）与解决方法。
