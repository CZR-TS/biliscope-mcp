## B站 Cookie 过期周期参考

根据社区经验和 B站 API 的设定，Cookie 的有效期通常如下：

1. **SESSDATA (核心会话)**：
   - 通常有效期为 **半年 (180天)**。
   - 但是，如果你在浏览器中手动点击了“退出登录”，该 Cookie 会立即失效。
   - 如果你在多台设备或多个浏览器频繁登录，旧的 SESSDATA 可能会被 B站的安全机制强制提前下线。

2. **bili_jct (CSRF Token)**：
   - 随 SESSDATA 同时生成，有效期基本一致。

3. **过期原因常见场景**：
   - **定期检查**：B站有时会触发安全验证，要求重新滑动验证码，此时即便 Cookie 未到 180 天也可能需要重新获取。
   - **IP 变动**：如果请求所在的服务器 IP 频繁跨地区变动，触发风控的概率会增加。
   - **多端登录**：B站对同时在线的 Web 端数量有限制。
